**Stufe A | maximal 2 Dateien | Ziel: 2 Dateien | keine Schema-Arbeit | keine neuen Seiten | keine robots.txt-Änderung | 48h Pause nach Deploy obligatorisch**

---

## Phase 0: Preflight-Sync (PFLICHT vor jeder Änderung)

Bevor du irgendetwas änderst, führe diese Schritte aus und gib das Ergebnis aus:

```bash
git branch --show-current
git rev-parse HEAD
git status -sb
git fetch origin
git log --oneline HEAD..origin/main
git log --oneline origin/main..HEAD
git diff --stat origin/main
```

**Erwartung:**
- Aktiver Branch ist `main` oder ein sauberer Arbeitsbranch
- `git log --oneline HEAD..origin/main` zeigt keine ungezogenen Commits (sonst: lokal hinter origin/main → erst pullen)
- `git log --oneline origin/main..HEAD` zeigt keine ungepushten Commits, die nicht zu D16 gehören
- `git diff --stat origin/main` zeigt keine relevanten Abweichungen
- `git status -sb` zeigt keine lokalen Änderungen, ungestagten Änderungen oder ungetrackten Änderungen in `client/public/llms.txt` oder `netlify.toml`

**Stoppe und melde, wenn:**
- lokale Änderungen, ungestagte Änderungen oder ungetrackte Änderungen in `client/public/llms.txt` oder `netlify.toml` vorliegen
- der lokale Stand mehrere Commits hinter `origin/main` liegt
- Branch-Divergenz besteht (sowohl lokale als auch Remote-Commits unbekannt)
- du auf einem unklaren Branch bist

Ohne sauberen Sync keine Änderungen ausführen.

## Datei 1 von max. 2: `netlify.toml`

### Aufgabe — drei konditionale Prüfungen

Lies den aktuellen Inhalt von `netlify.toml`. Führe drei unabhängige Prüfungen durch und reagiere konditional:

#### Prüfung A: Redirect-Ziel `/waermedaemmung`

Suche nach den Blöcken für `/waermedaemmung` und `/waermedaemmung/`.

| Vorgefundener Stand | Aktion |
|---|---|
| `to = "/dachwartung-bochum"` | Korrigieren auf `/aufsparrendaemmung-bochum` UND `force = true` ergänzen |
| `to = "/aufsparrendaemmung-bochum"` ohne `force = true` | Nur `force = true` ergänzen |
| `to = "/aufsparrendaemmung-bochum"` mit `force = true` | Keine Änderung |
| Kein Eintrag vorhanden | Beide Blöcke neu anlegen (siehe Sollzustand unten) |

**Sollzustand nach D16:**

```toml
[[redirects]]
  from = "/waermedaemmung"
  to = "/aufsparrendaemmung-bochum"
  status = 301
  force = true

[[redirects]]
  from = "/waermedaemmung/"
  to = "/aufsparrendaemmung-bochum"
  status = 301
  force = true
```

Wichtig: Beide Blöcke müssen **vor** dem SPA-Fallback-Eintrag (`from = "/*"` → `/index.html`) stehen. Die Reihenfolge der übrigen Redirects nicht ändern.

#### Prüfung B: Verwaister `[[edge_functions]]`-Block

Suche nach einem Block dieser Form:

```toml
[[edge_functions]]
  path = "/*"
  function = "og-inject"
```

| Vorgefundener Stand | Aktion |
|---|---|
| Block vorhanden | Block ersatzlos entfernen (og-inject ist laut Memory inaktiv, der Eintrag ist verwaist) |
| Block nicht vorhanden | Keine Änderung |
| Block mit anderem Funktionsnamen vorhanden | Stoppen, melden, nicht löschen |

#### Prüfung C: Keine anderen Redirects ändern

Alle übrigen Redirects, der `[build]`-Block und der SPA-Fallback bleiben unverändert.

## Format der URL-Liste

Markdown-Linkliste mit Kurzbeschreibung pro Eintrag (Spec-Format laut llmstxt.org). Beispiel:

```markdown
- [Dachsanierung Bochum](https://www.rex-bedachung.de/dachsanierung-bochum): Hub-Seite für Dachsanierung, Dämmung, Dachfenster und Klempnerei.
```

Keine reinen Klartext-URLs. Keine Doppellistungen.

**Diese Regel gilt für die Sektion `## Wichtige Seiten`.**

**In den Sektionen `## Referenzbilder VELUX` und `## Offizielle Profile` dürfen bestehende Roh-URLs erhalten bleiben, wenn sie als Bild- oder Profil-Metadaten dienen.**

## Go-/No-Go-Kriterien

**Go**, wenn:
- D15 mindestens 48 Stunden live ist (frühestens 30.04.2026)
- Preflight-Sync sauber durchläuft
- keine Ranking-/Indexierungs-Auffälligkeiten seit D15 sichtbar sind
- D16 maximal 2 Dateien ändert
- Build sauber läuft
- alle grep-Prüfungen erwartete Ergebnisse liefern
- Route-Check keine fehlenden Routen meldet

**No-Go**, wenn:
- D15 noch keine 48 Stunden live ist
- Preflight-Sync zeigt Branch-Versatz, ungezogene Commits oder lokale Änderungen in den Zieldateien
- `robots.txt` angefasst werden müsste
- der Agent zusätzliche Seiten, Komponenten oder Schema-Dateien ändern will
- mehr als 2 Dateien betroffen sind
- eine der erwarteten Routen nicht in `App.tsx` registriert ist
