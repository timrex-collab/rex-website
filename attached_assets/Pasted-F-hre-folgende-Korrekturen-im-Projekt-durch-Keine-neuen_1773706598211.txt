FIX 3 – Breadcrumb-Konsistenz über alle Seiten
Problem: Einige Seiten zeigen Startseite → Leistungen → [Seite], andere nur Startseite → [Seite]. Das BreadcrumbList-Schema folgt derselben Inkonsistenz.
Aktion: Vereinheitliche auf das einfachere Muster ohne /leistungen-Zwischenschritt:
  Startseite → [Seitenname]
Gehe folgende Seiten durch und passe jeweils an:
- Sichtbares Breadcrumb-UI (falls vorhanden)
- BreadcrumbList JSON-LD im Helmet-Block

Ziel-Schema für jede betroffene Seite:
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Startseite",
      "item": "https://www.rex-bedachung.de/"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "[Seitenname auf Deutsch, z.B. Dachreparatur Bochum]",
      "item": "https://www.rex-bedachung.de/[slug]"
    }
  ]
}

Betroffene Dateien (alle Seiten mit abweichendem Breadcrumb prüfen):
client/src/pages/Dachreparatur.tsx
client/src/pages/DachwartungBochum.tsx
client/src/pages/SteildachBochum.tsx

Falls eine Seite kein BreadcrumbList-Schema hat, füge es im Helmet-Block hinzu.
